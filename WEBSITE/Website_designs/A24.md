# Design System Inspired by A24

## 1. Visual Theme & Atmosphere

A24's design system embodies deliberate minimalism with cinematic sophistication. The interface prioritizes stark contrast between pure black and white, creating a gallery-like environment where content becomes the focal point. Every element serves narrative purpose—sparse whitespace, bold typography, and restrained motion align with the brand's art-house sensibility. The design conveys prestige through simplicity: no unnecessary ornamentation, clean geometric forms, and a commitment to legibility and impact. This is a studio's visual language: uncompromising, confident, and unapologetically bold.

**Key Characteristics**
- **Extreme contrast**: Pure black and white as primary palette with minimal mid-tones
- **Cinematic hierarchy**: Typography-driven layouts with generous negative space
- **Restrained ornamentation**: No gradients, no shadows (except subtle dropdowns), zero decorative elements
- **Precision typography**: Deliberate font sizing creates visual rhythm and content structure
- **Accessible minimal borders**: Subtle neutral lines guide visual flow without visual noise
- **Lowercase brand identity**: Typography as primary branding tool; A24 logo as anchor

## 3. Typography Rules

### Font Family
**Primary**: NB International Web (all weights: 400, 500, 700) — sans-serif, geometric, distinctive letterforms with balanced proportions.
**Fallback stack**: `NB International Web, -apple-system, BlinkMacSystemFont, Segoe UI, Helvetica, Arial, sans-serif`

### Hierarchy

| Role | Font | Size | Weight | Line Height | Letter Spacing | Notes |
|------|------|------|--------|-------------|-----------------|-------|
| Display | NB International Web | 74px | 500 | 68.08px | 0px | Hero typography for film titles, major headings. Used for impact. |
| Heading 1 | NB International Web | 15px | 400 | 25px | 0px | Page section headers, modal titles. Regular weight maintains elegance. |
| Heading 3 | NB International Web | 14px | 700 | 14px | 0px | Card titles, subsection headers. Bold for emphasis in compact spaces. |
| Heading 6 | NB International Web | 12.1px | 400 | 16px | 0px | Labels, metadata, fine print. Smallest heading tier. |
| Body | NB International Web | 18px | 400 | 23.94px | 0px | Primary content text. High legibility at this scale. |
| Caption / Small Text | NB International Web | 12px | 400 | 15.6px | 0px | Form labels, timestamps, footer content. Compact but readable. |
| Form Input Placeholder | NB International Web | 15px | 400 | normal | 0px | Input field text and hints. Matches heading 1 size for consistency. |
| Button Text | NB International Web | 16px | 700 | 22.4px | 0px | All CTA buttons. Bold weight signals interactivity. |

### Principles
- **Geometric clarity**: NB International Web's distinctive forms prioritize recognition over decoration.
- **Hierarchical contrast via weight**: 400-weight for body/context, 500-weight for large displays, 700-weight for buttons and emphasis.
- **Line-height generosity**: Above-standard line heights (1.33–1.4x multipliers) improve readability on light and dark backgrounds.
- **No letter-spacing variations**: Maintain typographic purity; default `0px` ensures font designer's intent.
- **Size jumps create rhythm**: 12px → 14px → 15px → 18px → 74px progression guides visual navigation.
- **Case sensitivity**: Lowercase and title-case used intentionally per context (brand is lowercase "a24").

## 5. Layout Principles

### Spacing System

**Base unit**: `4px`

**Scale**: `4px, 8px, 12px, 16px, 20px, 24px, 28px, 32px, 36px, 40px, 44px, 48px`

**Usage Context**:
- **Micro** (`4px`, `8px`): Icon padding, inline text spacing, tight form group spacing.
- **Small** (`12px`, `16px`): Button padding, form label margins, small card padding.
- **Medium** (`20px`, `24px`): Section padding, card-to-card gutters, heading margins.
- **Large** (`28px`, `32px`): Container padding, major section spacing, grid row/column gaps.
- **Extra Large** (`36px`, `40px`, `44px`, `48px`): Hero spacing, page-level vertical rhythm, modal padding.

**Application**: All margins and padding derive from this scale for visual consistency. Vertical rhythm uses multiples of the base unit to maintain alignment across the grid.

### Grid & Container

- **Max-width**: `1440px` (desktop content container)
- **Gutter width**: `32px` (horizontal padding on full-width containers)
- **Column count**: Flexible 12-column grid for desktop; 4-column on tablet; 1-column on mobile.
- **Section patterns**:
  - Full-width hero with centered content block (`max-width: 1000px`, centered horizontally).
  - Two-column layout for film lists and product grids (50/50 split on desktop; stacked on tablet/mobile).
  - Three-column footer section (contact, links, social).

### Whitespace Philosophy

A24's layout prioritizes negative space as a design element. Large padding and margins between sections create breathing room and draw focus to content. Vertical spacing is generous (48px–64px between major sections) to emphasize hierarchy and allow user visual rest. Horizontal padding ensures text never feels cramped (minimum 24px on mobile, 48px on desktop). This whitespace philosophy reinforces the gallery aesthetic: each piece of content is treated as a discrete artwork demanding isolated attention.

### Border Radius Scale

- **No radius**: `0px` (all components: buttons, inputs, cards, containers). This is fundamental to A24's geometric language—sharp corners convey precision and modern minimalism.

## 6. Depth & Elevation

| Level | Treatment | Use |
|-------|-----------|-----|
| Flat (Default) | `box-shadow: none` | All page content, buttons, standard containers, cards on white backgrounds. |
| Dropdown/Overlay | `box-shadow: rgba(0, 0, 0, 0.024) 2.1px 0px 2.2px 0px, rgba(0, 0, 0, 0.035) 5.1px 0px 5.3px 0px, rgba(0, 0, 0, 0.043) 9.5px 0px 9.9px 0px, rgba(0, 0, 0, 0.055) 17px 0px 17.6px 0px, rgba(0, 0, 0, 0.067) 31.8px 0px 33px 0px, rgba(0, 0, 0, 0.09) 76px 0px 79px 0px` | Modal overlays, dropdown menus, floating navigation elements. Shadow is subtle and horizontal-heavy. |
| Focus/Interaction | Outline or border shift | Input focus states use `2px solid #FFFFFF` outline. Button hover uses opacity or border color change rather than shadow. |

**Shadow Philosophy**: A24 eschews deep shadows and layered elevation in favor of flat, bordered components. The single shadow variant used is deliberately subtle—it suggests depth without visual distraction. This aligns with the brand's anti-skeuomorphism stance: interfaces are graphic, not tactile. Focus states rely on color and stroke changes rather than spatial depth.

## 7. Do's and Don'ts

### Do

- **Use pure `#000000` and `#FFFFFF` as primary colors** in all high-contrast interactive moments.
- **Maintain sharp 90-degree corners** (`0px` border-radius) across all components for geometric consistency.
- **Employ generous vertical spacing** (48px–64px minimum between sections) to reinforce the gallery aesthetic.
- **Prioritize typography as the primary design tool**—let font size, weight, and color carry hierarchy before introducing decoration.
- **Use NB International Web exclusively** for all text; never substitute fonts mid-project.
- **Align all elements to the 4px base grid** for padding, margins, and dimensions.
- **Keep border weights at `1px`**; never exceed this for primary UI elements.
- **Test all interactive states** (hover, active, focus, disabled) with explicit color and opacity shifts—no shadows for feedback.
- **Employ the subtle dropdown shadow only for modals and overlays**; never on standard cards or buttons.
- **Maintain a 2:1 aspect ratio or wider for hero images** to emphasize cinematic presentation.
- **Use the neutral grays judiciously**—reserve `#888888` and `#646464` for secondary content only.

### Don't

- **Never use rounded corners or soft transitions** (`border-radius > 0`); this contradicts A24's geometric identity.
- **Avoid drop shadows on standard UI components**; shadows are reserved for overlays alone.
- **Don't apply gradients or color overlays** without explicit design approval; A24's palette is flat and intentional.
- **Never introduce decorative icons or illustrations** without narrative purpose; minimalism extends to ornamentation.
- **Avoid light gray text (`#CACACA`, `#C2C2C2`) on dark gray backgrounds**—insufficient contrast violates accessibility standards.
- **Don't use the jade green (`#32AE88`) liberally**—its scarcity (1 instance total) is strategic; over-use dilutes impact.
- **Never exceed the max-width of `1440px`** on desktop; content must not stretch to fill entire screen on ultra-wide displays.
- **Avoid stacking more than 3 levels of heading hierarchy** in a single view; simplify navigation and information architecture.
- **Don't animate elements with slow, easing transitions**; if motion is introduced, it must be immediate and purposeful (100–200ms).
- **Never rely solely on color to convey information**; always pair color states with text labels or icon changes for accessibility.
- **Avoid centering large blocks of body text**; left-align for readability, with max-width of `600px–700px` per line.

## 8. Responsive Behavior

### Breakpoints

| Name | Width | Key Changes |
|------|-------|-------------|
| Mobile | 320px–767px | Single column layout, full-width inputs (100% minus 24px padding), stacked navigation, `16px` base font for body, `48px` button heights, reduced modal max-width to 90vw. |
| Tablet | 768px–1023px | Two-column grid for content, navigation shifts to hamburger menu, container gutter reduces to `24px`, inputs sized at `calc(50% - 12px)` for side-by-side form layouts. |
| Desktop | 1024px–1439px | Full three/four-column grids enabled, horizontal navigation visible, container max-width `1200px`, input widths at `325px` standard. |
| Large Desktop | 1440px+ | Max-width capped at `1440px`, content centered, generous side margins (auto), desktop-optimized two-column layouts. |

### Touch Targets

- **Minimum touch target size**: `48px × 48px` (buttons, navigation links, icon buttons).
- **Generous spacing**: `16px` minimum gap between adjacent interactive elements on mobile to prevent mis-taps.
- **Mobile button sizing**: All CTA buttons scale to full-width with `48px` height on viewports below `600px`.
- **Form inputs**: Minimum `44px` height on mobile (increased from desktop `30px–34px` for touch comfort).
- **Icon buttons**: Never smaller than `32px × 32px` on mobile; desktop can go to `24px × 24px` for navigation.

### Collapsing Strategy

- **Navigation**: Desktop horizontal menu collapses to stacked hamburger on tablet/mobile; flyout drawer slides from left at `0px` z-index over content.
- **Two-column layouts**: Side-by-side product/film grids collapse to single column below `768px`; images stack vertically.
- **Form fields**: Inline form groups (e.g., email + button) reflow to full-width stacked inputs on mobile; button width matches input width.
- **Modal/Overlay**: Desktop modal max-width `600px` reduces to `90vw` on mobile with `12px` side margins for breathing room.
- **Typography scaling**: `74px` display heading reduces to `36px` on tablet, `24px` on mobile; `18px` body remains stable to `16px` minimum on mobile.
- **Padding/margins**: Desktop `48px` section padding reduces to `32px` on tablet, `24px` on mobile to maximize content area.
- **Grid gutters**: Desktop `32px` gutter reduces to `24px` on tablet, `16px` on mobile.

- **Accent (Reserved)**: Jade Green (`#32AE88`) — use sparingly
- **Modal Background**: Black (`#000000`)

### Iteration Guide

1. **Typography first**: All sizing and weight from the hierarchy table; NB International Web is non-negotiable; use 400/500/700 weights only.

2. **Color contrast**: Every text element must achieve WCAG AA minimum (4.5:1 for body, 3:1 for large text). Test black-on-white and white-on-black combinations.

3. **Spacing grid**: All margin and padding values must be multiples of `4px`. Vertical rhythm uses `24px`–`48px` increments between major sections.

4. **Zero border-radius**: Every component—buttons, inputs, cards, modals—has `border-radius: 0px`. No exceptions.

5. **Border-driven UI**: Interaction feedback relies on `1px` borders (`#000000` / `#FFFFFF` contextual) and color shifts, never shadows or alpha transparency (except transparent backgrounds for ghost buttons).

6. **Shadow restraint**: Only dropdown/overlay shadow variant applied (`box-shadow: rgba(0, 0, 0, 0.024) 2.1px 0px 2.2px 0px, ...`). No other shadows on default components.

7. **Input styling**: All inputs use `rgba(0, 0, 0, 0)` transparent background with `1px` border. Padding is `22px 18px` for standard inputs; `0px 0px 8px 0px` for large display inputs.

8. **Button states**: Primary/secondary buttons use solid fills (`#FFFFFF` / `#000000`) with `700` font-weight `16px` text. Ghost buttons remain transparent with color-only state changes.

9. **Max-width adherence**: Container max-width capped at `1440px` on desktop; responsive scaling below per breakpoint table. Full-width containers include `32px` gutter padding.

10. **Focus/accessibility**: Input focus states use `outline: 2px solid #FFFFFF` offset `2px`; visible focus indicators on all interactive elements; never hide default focus outlines.
