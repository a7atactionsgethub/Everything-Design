GROUNDED FOOTER GENERATION MODE

Design a footer that feels like the physical foundation and terminal layer of the website experience.

The footer should create a sense of visual gravity — as if the entire interface settles naturally into it.

This is not a decorative ending section.
It is the architectural base of the digital environment.

The footer must feel:
- grounded
- anchored
- weighty
- stable
- cinematic
- spatially disciplined
- calm
- structurally intentional

Before generating:
- analyze the current project's visual identity
- study spacing rhythm and layout density
- understand typography hierarchy
- detect the structural logic of the interface
- preserve the project's existing visual philosophy

The footer should:
- visually lower the energy of the experience
- create emotional closure
- feel horizontally stable
- use strong compositional balance
- act as a base platform for the interface

Focus heavily on:
- visual weight distribution
- negative space
- structural alignment
- controlled density
- calm typography
- architectural composition
- baseline consistency
- premium restraint
- heavy horizontal framing
- subtle separation layers

Avoid:
- startup-style footers
- floating/glassy aesthetics
- excessive CTA sections
- overexposed social icons
- crowded multi-column layouts
- trendy gimmicks
- decorative clutter
- aggressive animations
- visual noise
- disconnected sections

Typography should feel:
- deliberate
- restrained
- grounded
- confident
- spatially balanced

Interactions should:
- remain subtle and nearly invisible
- reinforce stability instead of excitement
- feel smooth, dense, and premium

The footer must feel integrated into the site's architecture, as if it was always part of the system rather than appended afterward.

Do not redesign the website.
Do not introduce unrelated styling systems.
Do not break the established visual language.

Output:
1. Footer concept direction
2. Structural composition
3. Visual weight strategy
4. Typography treatment
5. Spacing and density logic
6. Responsive behavior
7. Interaction philosophy
8. Architectural integration notes
